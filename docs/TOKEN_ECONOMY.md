# Правила экономии токенов

Документ для Claude Code — как работать в этом репозитории эффективно.

## Отключённые плагины

Не использовать, не требовать:
- superpowers, context7, code-review, code-simplifier
- claude-md-management, security-guidance, chrome-devtools-mcp

Активны: `claude-mem`, `caveman`.

## Чтение репозитория

- Не читать весь репозиторий без необходимости.
- Читать только файлы, релевантные текущей задаче.
- Использовать `grep`/`find` вместо `cat` для поиска.
- Для ошибок: `tail -50 file.log`, не весь файл.

## Рабочий процесс

1. Получить задачу → написать короткий план (3–5 пунктов).
2. Согласовать с пользователем.
3. Реализовать по конкретным файлам.
4. Обновить `docs/SESSION_LOG.md`.

## Формат отчёта после этапа

Показывать только:
- Изменённые файлы (список)
- Что сделано (1–3 строки)
- Как проверить
- Какие тесты запускать
- Что делать дальше

Не пересказывать код в ответе.

## Запреты в коде

- Монолитный скрипт > 500 строк — запрещён.
- Смешивать broker client, strategy, Telegram, DB в одном файле — запрещён.
- Хардкодить токены — запрещён.
- Логировать токены — запрещён.

## Claude Mem

Использовать для фиксации прогресса между сессиями.  
После каждого этапа: сохранить короткую запись в claude-mem.
